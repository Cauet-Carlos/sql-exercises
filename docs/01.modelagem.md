# Modelagem do Banco de Dados

O banco de dados utilizado neste repositório de estudos sobre sql simula um cenário de e-commerce, permitindo a prática de consultas analíticas utilizando SQL.

A estrutura foi construída para possibilitar análises de:
- vendas;
- clientes;
- produtos;
- faturamento;
- métricas temporais;
- comportamento de compra.

---

# Estrutura das Tabelas

## clientes

Tabela responsável pelo armazenamento das informações dos clientes.

Principais campos:
- id_cliente
- nome
- email

---

## pedidos

Tabela que registra os pedidos realizados pelos clientes.

Principais campos:
- id_pedido
- id_cliente
- data_pedido
- status

---

## produtos

Tabela contendo os produtos disponíveis no e-commerce.

Principais campos:
- id_produto
- nome
- preco

---

## itens_pedido

Tabela relacional responsável por conectar pedidos e produtos.

Principais campos:
- id_item
- id_pedido
- id_produto
- quantidade
- preco_unitario

---

# Relacionamentos

## clientes -> pedidos

Um cliente pode possuir vários pedidos.

Relacionamento:
1:N

---

## pedidos -> itens_pedido

Um pedido pode conter vários itens.

Relacionamento:
1:N

---

## produtos -> itens_pedido

Um produto pode aparecer em vários pedidos.

Relacionamento:
1:N

---

# Objetivo da Modelagem

A modelagem foi criada com foco em prática analítica utilizando SQL, permitindo exercícios envolvendo:

- JOINs
- agregações
- análises temporais
- subqueries
- CTEs
- window functions
- métricas de negócio

---

# Cenários Simulados

Durante os exercícios foram realizadas análises como:

- faturamento mensal;
- ticket médio;
- crescimento percentual;
- ranking de clientes;
- segmentação por faturamento;
- média móvel;
- acumulados;
- produtos mais vendidos.
